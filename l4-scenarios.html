📌 Level 04 · Modelling · L4.06 — Scenarios, Sensitivities & Stress Tests

A financial model with one set of assumptions is a point estimate — it gives a single answer to a question that has an infinite number of possible answers. Scenarios and sensitivities transform a single output into a range, and stress tests reveal where the investment thesis breaks. These are not optional features — they are mandatory for any model presented to clients or used in credit decisions.

Building a Scenario Toggle

Step 1: Create a Scenarios tab with three columns: Assumption Base Upside Downside Revenue growth Y1 10% 14% 5% Revenue growth Y2 9% 12% 3% EBITDA margin 32% 35% 27% Churn rate 8% 6% 14% Step 2: Add a dropdown in the Assumptions tab: Cell A1: [dropdown] Base / Upside / Downside Step 3: Reference with CHOOSE or INDEX/MATCH: Revenue growth Y1 = CHOOSE(scenario_num, 10%, 14%, 5%) Or: = INDEX(scenarios_table, row, MATCH(A1, scenario_names, 0)) Step 4: All model forecasts auto-update when dropdown changes

Stress Test — Finding the Break-Even

For a lender: "At what EBITDA level does interest cover breach 2.0x?" Interest Cover = EBITDA / Interest Expense Set IC = 2.0x, solve for EBITDA: Break-even EBITDA = 2.0 × Interest Expense If current EBITDA = £60m and interest = £20m: Current IC = 3.0x Break-even EBITDA = 2.0 × £20m = £40m Headroom = (£60m − £40m) / £60m = 33% EBITDA decline before breach For PE: "What exit multiple generates at least 20% IRR?" Goal Seek: set IRR cell = 20%, by changing exit multiple cell Result: minimum exit multiple required for target return
Present Ranges, Not Point Estimates

No client presentation should show a single EV or IRR number without a sensitivity range. The sensitivity table is the honesty mechanism — it shows the reader that the "base case" is one point in a wide possible range. A DCF with WACC sensitivity showing EV of £380m–£620m is far more informative and defensible than a single EV of £500m. Clients who understand finance expect the range. Clients who don't are better served by seeing it.

🔑 Lender's Perspective — JaceyTrayd

We run three mandatory scenarios on every credit: Base, Downside (EBITDA −20%), and Severe (EBITDA −35%). We set covenants based on the Downside scenario — the company must be able to service debt even under material deterioration. We check: in the Severe scenario, does the company still avoid insolvency? Does it breach covenants before we can take action? The Severe scenario determines whether we are comfortable with the risk — not the Base. We never lend based on Base alone.
