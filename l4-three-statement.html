📌 Level 04 · Modelling · L4.02 — The 3-Statement Model

The 3-statement model is the engine of all IB financial analysis. Every valuation, every deal model, every credit assessment is built on top of it. If the three statements do not link correctly, every downstream output — IRR, EV, implied share price — is wrong.

The Build Sequence — always in this order

StepBuildKey outputFeeds into
1Income Statement (P&L)Net IncomeCFS starting line + BS retained earnings
2Debt ScheduleEnding debt balances, interest expenseP&L interest line (circularity), BS debt balances, CFS debt drawdowns
3Working Capital ScheduleAR, AP, Inventory balancesBS current assets/liabilities, CFS WC changes
4Capex / PP&E ScheduleNet PP&E, D&A chargeP&L D&A, BS PP&E, CFS capex outflow
5Cash Flow StatementNet change in cashBS cash line (opening + net change = closing)
6Balance SheetTotal Assets = Liabilities + Equity (must balance)Final check — surfaces all errors

The Five Critical Linkages

Link 1: Net Income → CFS (starting line of indirect method) Link 2: Net Income → Retained Earnings (BS equity section) RE(end) = RE(start) + Net Income - Dividends Link 3: Net change in cash (CFS) → Cash (BS) Cash(BS, end) = Cash(BS, start) + Net Change (CFS) Link 4: Capex (CFS) → PP&E (BS) | D&A (P&L) → Accum. Dep. (BS) Net PP&E = Gross PP&E - Accumulated Depreciation Link 5: Debt drawn/repaid (CFS Financing) → Debt balance (BS Liabilities) Ending Debt = Opening Debt + Drawdowns - Repayments

Handling the Circularity

Interest expense depends on the debt balance. The debt balance depends on how much cash the company has. Cash depends on net income after interest. Net income depends on interest expense. This loop means the statements cannot be solved sequentially.

Professional solution: Use a circularity switch — an input cell (call it CIRC_SWITCH) set to 0 or 1. When 0: interest formula references prior year's debt balance (breaks the loop, model is non-circular). When 1: interest references the current year's solved debt balance (circular, requires iterative calculations). Build the model with the switch at 0 first. Then turn it to 1 and enable iterative calculations to solve. This lets you toggle the circularity on/off for debugging.

The Balance Check — your most important formula

In the Balance Sheet tab, final row: =ROUND(Total Assets - (Total Liabilities + Total Equity), 0) Format this cell: Green background if = 0 (model balances) Red background if ≠ 0 (something is broken) Use Excel conditional formatting: Rule 1: IF(check=0, fill green) Rule 2: IF(check≠0, fill red) NEVER submit a model with a red balance check. If it is red, work backwards through the five links until you find the break.
The Retained Earnings Roll — Where Most BS Breaks Originate

The most common source of balance sheet imbalance is the retained earnings roll. Retained earnings at close = retained earnings at start + net income − dividends. If you accidentally include any other adjustment — a share issuance, a revaluation, a one-off — without reflecting it correctly in equity, the BS breaks. Build the retained earnings formula first, then check it carefully before building anything else in equity.

In a Real Bank

On a live deal, the 3-statement model is typically reviewed by three layers: the analyst (builds it), the associate or VP (checks it), and the MD (uses it in the client meeting). Each layer has a different focus: the analyst checks that formulas are correct; the associate checks that assumptions are defensible; the MD checks that the outputs make strategic sense. An analyst who builds a clean, clearly structured model with a prominent balance check makes everyone else's job easier — and gets noticed for it.

🔑 Lender's Perspective — JaceyTrayd

When we receive a borrower's model, the first thing our credit team does is run a structural audit: does the balance sheet balance? Does the cash tie? Does the debt schedule reconcile to the balance sheet? A model that fails these basic checks within the first five minutes tells us the advisors cut corners — and if they cut corners on the model, we ask what else was done carelessly. Model quality is a proxy for the overall diligence quality of the deal team.
